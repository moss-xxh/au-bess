## 🔧 P0级具体执行指令 (基于Gemini 2.0修正案)

### 第一步: 物理清理与静默皮肤 (P0)
```bash
# 目标文件：components/header-nav.js
# 1. 保留第21行 theme-dark 注入 (不删除)
# 2. 删除第126行的themeToggleBtn变量定义
sed -i '126s/.*//' components/header-nav.js

# 3. 删除第154-162行的headerRight事件委托逻辑
sed -i '154,162d' components/header-nav.js

# 验证皮肤逻辑清理
grep "themeToggle" components/header-nav.js || echo "✅ themeToggle已清理完成"
```

### 第二步: 报表重构 (P0)
```bash
# 在 /var/www/bess-v2/ 目录执行
cd /var/www/bess-v2/

# 动作1: 执行文件重命名
mv 002.html reports.html

# 动作2: 修改header-nav.js第87行的引用路径
sed -i "87s|href: '002.html'|href: 'reports.html'|g" components/header-nav.js

# 动作3: i18n标签替换(基于现有键值)
# 将reports.html第122, 127行等硬编码汉字替换为data-i18n标签
# 重点：使用report.compare.day等现有键名，禁止发明新键值
sed -i 's|比昨日|<span class="compare-text" data-i18n="report.compare.day"></span>|g' reports.html
sed -i 's|累计收益|<span data-i18n="report.stats.charge"></span>|g' reports.html

# 验证文件重命名
grep -r "002\.html" . --exclude-dir=.git || echo "✅ 002.html引用已清理完成"
```

### 第三步: 角色页面解耦 (P1)
```bash
# 在AU-03根目录创建角色目录结构
mkdir -p pages/operator pages/owner

# 创建角色专用页面(从v2搬运并定制)
# operator/station.html: 强调"实时调度"数据
cp station.html pages/operator/station.html
# owner/station.html: 强调"资产统计"数据  
cp station.html pages/owner/station.html

# 同样处理fault-alarm.html
cp fault-alarm.html pages/operator/fault-alarm.html
cp fault-alarm.html pages/owner/fault-alarm.html

# 更新header-nav.js中的路径映射
# 运维角色指向: pages/operator/
# 业主角色指向: pages/owner/
```

### 第四步: 最终部署到AU-03
```bash

cd /tmp && rm -rf au03-deploy && mkdir au03-deploy && cd au03-deploy
```
