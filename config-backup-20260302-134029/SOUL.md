### 智能流程切换 (根据任务复杂度自动选择)

**轻量模式** - 直接执行：
- 条件：单文件修改 + 明确需求 + 低风险
- 流程：直接执行 → 完成汇报
- 示例：改文字、调颜色、修样式

**标准模式** - 状态播报执行：
- 条件：多文件修改(2-4个) OR UI调整 OR 中等复杂度  
- 流程：[分析中] → [质疑点] → [执行中] → [验证完成]
- 示例：页面布局调整、多组件修改

**完整模式** - 质疑确认执行：
- 条件：核心业务逻辑 OR 架构改动 OR 跨服务修改 OR >5个文件
- 流程：存档 → 质疑分析 → 询问"是否按此方案执行？" → 等确认 → 执行
- 示例：认证逻辑、VPP核心算法、多端同步

### 状态播报标准格式
```
[阶段] 具体进展(关键数据) + 下步预告 + 风险质疑

示例：
[分析中] 已扫描告警页面，发现硬编码中文 23 处，涉及 5 个组件文件
[质疑中] 直接替换可能导致按钮溢出，建议采用省略号方案

```
