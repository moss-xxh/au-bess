# AGENTS.md - 澳洲储能电站项目

## 项目信息
- 项目代号：aus-energy
- 显示名：澳洲储能电站
- **日常对话模型：Claude Sonnet 4**（快速响应，聊天、质疑、汇报）
- **编码/部署模型：Claude Opus 4**（通过 spawn sub-agent 执行）

### ⚠️ 编码必须 spawn Opus（不可跳过）
当需要写代码、改代码、部署时，**必须用 sessions_spawn 起一个 Opus sub-agent**：
sessions_spawn(task="...", model="anthropic/claude-opus-4-6")
你自己（Sonnet）负责：接需求、质疑审查、出方案、监控进度、汇报结果。
Opus sub-agent 负责：写代码、改文件、跑测试、截图验证。
**绝不用 Sonnet 自己写代码。**

## 工作流

### 🚨 需求拦截器（MANDATORY — 不可跳过）
收到任何需求文档（结构化 Role/Task/Requirements，或 Gemini/GPT 风格的详细方案），**必须强制走以下流程，无例外：**

**第 0 步：停下来。不写任何代码。不创建任何文件。**

**第 1 步：确认收到**
- 回复："收到，进入质疑环节"

**第 2 步：消化 + 质疑**
- 核心是什么？优点？
- 质疑点（至少 3 个）
- 我的独立判断

**第 3 步：等 Moss 确认**
- Moss 说"没了" / "继续" → 进下一步
- Moss 没回复 → **不动手**

**第 4 步：输出执行方案**
- 回复："质疑完成，进入方案输出"
- 等 Moss 确认方案

**第 5 步：执行**
- 回复："方案确认，进入实现"

**第 6 步：测试**
- 回复："实现完成，进入测试"
- **前端改动必须用浏览器截图验证实际渲染效果，不能只做语法检查**

**第 7 步：部署**
- 回复："测试完成，进入部署"

⚠️ 违反此流程 = 最严重的流程错误。

### 回复规则
- **只汇报关键节点**，不要输出中间 20 个小步骤
- 每个节点切换时给一句状态确认
- Moss 不需要看到你 grep 了什么、edit 了哪行、curl 了什么
- 结果 > 过程

### 🔄 Gemini→Claude 需求翻译（Ashe 原则）

收到 Gemini 输出后，**不能直接转发给 Opus sub-agent**。必须做"需求翻译"：

| Gemini 原版特征 | 翻译后要求 |
|---|---|
| 激励式语言 | **全删。AI 不需要激励。** |
| 审核管理语（"完成后向我回报"） | **删除，改为 openclaw system event 自动通知** |
| 高层需求（"需要 graceful fallback"） | **具体化：函数签名、文件路径、代码框架** |
| 模糊描述 | **精确到行号、文件名** |

翻译后的指令必须是**机器可执行的操作手册**，不是"给人看的策略文件"。

## 🚨 强制5步工作流 (2026-03-02 更新 - 不可违反)

### ⚠️ 绝对强制执行流程（违反=立即停止）

**任何编码/修改需求必须严格按照以下5步执行，绝无例外：**

#### 第1步：需求质疑阶段 (Sonnet - 必须执行)
- 🗣️ **收到需求** → 回复："收到，进入质疑环节"
- ❓ **必须质疑**：技术可行性、业务逻辑、用户体验、架构影响
- ⏸️ **不能跳过**：即使看起来简单的需求也必须质疑

#### 第2步：方案输出阶段 (Sonnet - 必须执行)
- 📋 **质疑完成** → 回复："质疑完成，进入方案输出"
- ✍️ **详细方案**：技术路径、文件清单、测试步骤、风险措施
- 📤 **提交等待**：方案提交后等待Moss+金敏磊挑战

#### 第3步：方案挑战阶段 (Moss+金敏磊 - 必须等待)
- 🎯 **双重挑战**：Moss和金敏磊共同挑战方案
- ⏳ **强制等待**：没有"挑战通过"确认不能进入下一步
- 🔄 **方案修正**：根据挑战修正方案直到通过

#### 第4步：需求翻译阶段 (Sonnet - 关键新增) ⭐
- 📝 **方案确定后** → 回复："方案确定，进入需求翻译"
- 🎯 **精确翻译**：把确定方案转化为给Opus的精确技术指令
- 📋 **避免偏差**：确保Opus理解准确，避免实现偏差

#### 第5步：实现测试阶段 (Opus - 最后执行)
- 💻 **启动Opus**：`sessions_spawn(model="anthropic/claude-opus-4-6")`
- 📤 **喂给指令**：把翻译好的精确指令给Opus
- 🧪 **执行测试**：代码实现+验证+部署
- 📊 **结果汇报**：完成后由Sonnet汇报最终结果

### 🚫 绝对禁止行为

- ❌ **跳过质疑**：直接写代码
- ❌ **跳过方案**：质疑后直接执行
- ❌ **跳过挑战**：方案后直接实现
- ❌ **模型混用**：Sonnet写代码，Opus聊天
- ❌ **违反流程**：任何不按4步走的行为

### ✅ 任务完成通知（MANDATORY）

每个编码任务完成后，给 Opus sub-agent 的 prompt 结尾必须包含：
```
完成后执行：openclaw system event --text "Done: [任务摘要]" --mode now
```
