# AGENTS.md - aus-energy 启动配置

## 每次启动

1. 读 `SOUL.md` — 5步工作流程（最重要）
2. 读 `USER.md` — Moss的偏好
3. 读 `IDENTITY.md` — 项目信息
4. 读 `memory/` 最新日志（今天+昨天）
5. 如果有 `MEMORY.md`，快速扫关键坐标

## 5步流程速查

1. **接收+质疑** → 回复"收到，正在分析质疑" → 输出质疑点
2. **方案输出** → 回复"质疑完成，输出方案" → 等Moss确认
3. **需求翻译** → 回复"方案确认，编写需求文档" → 存档 `memory/tasks/`
4. **实现代码** → 回复"文档存档，spawn Opus" → 监控执行
5. **测试验证** → 回复"实现完成，测试验证" → 汇报结果

## 需求翻译文档模板

存档路径：`memory/tasks/YYYY-MM-DD-<描述>.md`

```markdown
# 需求：<标题>
## 背景
<一句话说明为什么做>
## 修改清单
- 文件1：改什么
- 文件2：改什么
## 具体操作
<精确到行号的修改指令>
## 验证方法
<怎么确认改对了>
```

## Opus调度

给Opus的指令 = 第3步的需求翻译文档内容。
不要给废话背景，不要给Gemini原文，只给具体操作。

## 记忆管理

- 需求文档 → `memory/tasks/`
- 长期信息 → `MEMORY.md`
